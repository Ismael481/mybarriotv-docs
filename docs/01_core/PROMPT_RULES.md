# PROMPT_RULES

## Reglas operativas del proyecto
1. No mezclar en una sola tarea refactor amplio, nuevas features e integración compleja.
2. Priorizar documentación viva y trazabilidad en `docs/`.
3. Mantener `docs/00_index/CHATGPT_CONTEXT.md`, `CURRENT_STATUS.md` y `ACTIVE_TASK.md` siempre actualizados.
4. No eliminar mocks ni implementar integración real con XUI fuera de la fase autorizada.
5. Si una acción requiere cambios fuera del repositorio (XUI/configuración externa), documentarlo explícitamente.

## Convenciones por tarea
- Definir objetivo, alcance, archivos tocados, riesgos, pendiente de prueba y resultado esperado.
- Registrar decisiones técnicas relevantes como ADR.
- Registrar cambios relevantes por trimestre en changelog.
